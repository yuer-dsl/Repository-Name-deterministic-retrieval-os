# FAQ

### 1. Does this replace LLMs?
No. It replaces unstable retrieval and unstable reasoning around LLMs.

### 2. Is this better than RAG?
For stability and reproducibility: **yes**.  
For semantic breadth: RAG still has uses.

### 3. Does this guarantee correctness?
No. It guarantees **controllability**, not **truth**.

### 4. Does this protocol inspect model internals?
No. It only structures inputs and execution paths.
